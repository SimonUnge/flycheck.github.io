Appendix A Supported languages

This chapter lists the languages supported by Flycheck. Some 3rd party extensions add support for more languages:

  • flycheck-ledger adds a syntax checker for files of the Ledger accounting tool.
  • flycheck-mercury adds a syntax checker for the Mercury programming language.
  • flycheck-ocaml adds a syntax checker for the OCaml programming language, using the Merlin tool.

Each language has one or more syntax checkers, whose names follow a convention of ‘language-tool’ where language is the programming language checked by the checker, and tool the name of the checker tool.

The syntax checkers are listed in the order they are applied to a buffer. Use C-c ! ? (flycheck-describe-checker) to obtain more information about a syntax checker.

Ada

  • ada-gnat (GNAT)

AsciiDoc

  • asciidoc: AsciiDoc

C/C++

  1. c/c++-clang (syntax and type check with Clang) or c/c++-gcc (syntax and type check with GCC)
  2. c/c++-cppcheck (style and error check with cppcheck)

CFEngine

  • cfengine (CFEngine)

Chef

  • chef-foodcritic (style check with foodcritic)

Coffeescript

  1. coffee (syntax check with coffee)
  2. coffee-coffeelint (code style check with coffeelint)

Coq

  • coq (Coq)

CSS

  • css-csslint (style and error check with CSSLint)

D

  • d-dmd (DMD)

The extension flycheck-d-unittest adds a syntax checker which runs D unittests on the fly and reports the results.

Elixir

  • elixir (Elixir interpreter)

Emacs Lisp

  1. emacs-lisp (syntax check with the built-in Emacs byte compiler)
  2. emacs-lisp-checkdoc (code and documentation check with checkdoc)

The extension flycheck-package adds a syntax checker which checks for violations of Emacs Lisp packaging conventions.

Erlang

  • erlang (Erlang)

ERuby

  • eruby-erubis (erubis)

Fortran

  1. fortran-gfortran (GFortran)

Go

  1. go-gofmt (syntax check with gofmt)
  2. go-golint (coding style with Golint)
  3. go-vet (check for suspicious code with go tool vet)
  4. go-build or go-test (syntax and type check with Go, for source and tests respectively)
  5. go-errcheck (check for unhandled error returns with errcheck)

Haml

  • haml (Haml)

Handlebars

  • handlebars (Handlebars)

Haskell

  1. haskell-ghc (syntax and type checker with GHC)
  2. haskell-hlint (style checker with hlint)

The extension flycheck-haskell configures Flycheck from the current Cabal project, and adds support for Cabal sandboxes. The extension flycheck-hdevtools adds an alternative syntax checker for haskell-ghc using hdevtools.

HTML

  • html-tidy (Tidy HTML5)

Javascript

  • javascript-jshint (JSHint), or
  • javascript-eslint (ESLint), or
  • javascript-gjslint (Closure Linter)

JSON

  • json-jsonlint (jsonlint)

Less

  • less (less)

Lua

  • lua (Lua compiler)

Perl

  1. perl (syntax check with the Perl interpreter)
  2. perl-perlcritic (style and code check with Perl::Critic)

PHP

  1. php (syntax check with PHP CLI)
  2. php-phpmd (code check with PHP Mess Detector)
  3. php-phpcs (style check with PHP CodeSniffer)

Puppet

  • puppet-parser (syntax check with Puppet)
  • puppet-lint (style check with Puppet Lint)

Python

  • python-flake8 (syntax and style checking with flake8), or
  • python-pylint (syntax and style checking with Pylint), or
  • python-pycompile (syntax checking with Python’s built-in byte compiler, as last resort)

The extension flycheck-pyflakes adds a syntax checker using Pyflakes.

R

  • r-lintr (syntax and style check with lintr)

Racket

  • racket (Racket)

RPM Spec

  • rpm-rpmlint (rpmlint)

reStructuredText

  • rst (using docutils, for plain reStructuredFiles files), or rst-sphinx (using Sphinx, for Sphinx documentation files)

Ruby

  1. ruby-rubocop (syntax and style check using RuboCop)
  2. ruby-rubylint (syntax and style check using ruby-lint)

If none of the above is available, Flycheck will fall back to one of the following checkers for very basic checking:

  • ruby (using the standard Ruby interpreter),
  • ruby-jruby (using JRuby)

Rust

  • rust (using the Rust compiler)

The flycheck-rust extension configures Flycheck according to the current Cargo project.

Sass

  • sass (using the standard Sass processor)

Scala

  1. scala (syntax and type check using the Scala compiler)
  2. scala-scalastyle (style check using Scalastyle)

SCSS

  • scss (using the standard SCSS processor

Shell scripting languages

  1. Syntax check with one of the following checkers, depending on the shell used for the current Sh Mode buffer,
    • sh-bash (for Bash)
    • sh-posix-dash or sh-posix-bash (for POSIX shell scripts, using Dash or Bash respectively)
    • sh-zsh (for Zsh)
  2. sh-shellcheck (code and style check using ShellCheck)

Slim

  • slim (using Slim)

TeX/LaTeX

  • tex-chktex (style check using ChkTeX), or
  • tex-lacheck (style check using Lacheck)

Texinfo

  • texinfo (using makeinfo)

Verilog

  • verilog-verilator (using Verilator)

XML

  • xml-xmlstarlet (using XMLStarlet), or
  • xml-xmllint (using xmllint)

YAML

  • yaml-jsyaml (using js-yaml), or
  • yaml-ruby (using Ruby’s YAML parser)
